üêû Debugging Guide

This section explains how to systematically debug each major subsystem in your LIMO Autonomous System ‚Äî object detection, navigation, arm control ‚Äî and which diagnostic tools to use for fast root-cause analysis.

üß© How to Debug Each Subsystem‚Äã

1. Object Detection (object_detector.py)‚Äã

Goal: Verify YOLO is publishing /yolo/annotated, /object_found, and /target_pose.

Checks:

# Is the node running?
ros2 node list | grep yolo_nav_detector

# Is camera data arriving?
ros2 topic hz /camera/color/image_raw
ros2 topic hz /camera/depth/image_raw

# Is YOLO producing output?
ros2 topic echo /object_found -n 1
ros2 topic echo /target_pose -n 1

Key Log Lines to Expect:

‚úÖ YOLO detector node initialized with synchronized inputs.
[üìç DETECTED] <class> 2D(...) 3D(...)
[üåç POSE] Published /target_pose ‚Üí map(...)

If It Fails:

  • No detections ‚Üí check conf threshold in code.
  • Depth invalid ‚Üí verify /camera/depth/image_raw encoding is 16UC1.
  • TF errors ‚Üí confirm static_transform_publisher is launched before YOLO.

2. Navigation (Nav2 + mission_manager.py)‚Äã

Goal: Ensure robot can accept goals and move.

Checks:

# Nav2 actions exist
ros2 action list | grep navigate_to_pose

# Send a manual goal
ros2 action send_goal /navigate_to_pose nav2_msgs/action/NavigateToPose \
"{pose: {header: {frame_id: map}, pose: {position: {x: 1.0, y: 0.0}, orientation: {w: 1.0}}}}"

Key Log Lines:

üü¢ Exploration node ready.
üöÄ Navigating to: (x, y, yaw)
‚úÖ Goal accepted.
üèÅ Goal reached.

If It Fails:

  • ‚ùå Nav2 server not ready! ‚Üí Increase TimerAction delay before Mission Manager starts.
  • ‚ùå Goal rejected. ‚Üí Check map frame alignment and initial pose (pose_setter.py).

3. Arm Control (pick_node.py / drop_node.py)‚Äã

Goal: Confirm MyCobot arm moves through pick/drop sequences.

Checks:

# Directly run pick node
ros2 run mycobot_arm pick_node --ros-args -p m5_ip:=192.168.137.75

# Directly run drop node
ros2 run mycobot_arm drop_node --ros-args -p m5_ip:=192.168.137.75

Key Log Lines:

ü§ñ Connected to MyCobot over Wi-Fi
üß≤ Grabbed the object!
‚úÖ Pick sequence complete!
ü™£ Dropped the object!
‚úÖ Drop sequence complete!

If It Fails:

  • Socket errors ‚Üí check Wi-Fi hotspot SSID/password.
  • Motion off ‚Üí verify joint angles in code match your hardware setup.

Pro Tip

When debugging, run one subsystem at a time ‚Äî camera + YOLO, then Nav2 alone, then Mission Manager ‚Äî before launching full_system.launch.py.

üõ† Step-by-Step Debug Flow‚Äã

1. Camera First‚Äã

  • Launch only the camera node.
  • Verify RGB + depth topics are publishing at good rates.

2. Add YOLO‚Äã

  • Run yolo_detector.
  • Confirm detections and /target_pose output.

3. Add Nav2‚Äã

  • Start Nav2 with known map.
  • Publish initial pose manually or via pose_setter.py.
  • Send a test goal.

4. Add Mission Manager‚Äã

  • Confirm it receives /target_pose and interrupts exploration.

5. Test Arm‚Äã

  • Run pick/drop independently.
  • Check Wi-Fi IP detection from full_system.launch.py.

6. Full System‚Äã

  • Launch full_system.launch.py.
  • Watch logs for sequence: 
    YOLO ‚Üí Target Pose ‚Üí Nav2 ‚Üí Pick ‚Üí Return ‚Üí Drop

üß† Prevention Strategies During Debugging‚Äã

  • Use RViz to confirm robot pose, goals, and detections visually.
  • Save mission logs: 
    ros2 launch nav_handler full_system.launch.py | tee mission.log
  • Version your configs (map.yaml, nav2_params.yaml, TF static transforms).
  • Avoid overlapping launches ‚Äî don't start two nodes that publish the same topic.
